# Lab Basic 04

## Задание: анализ кода, памяти и результата

Ниже разбор каждого фрагмента:
- какие инструкции выполняются;
- сколько ячеек во временной памяти (стековые локальные переменные);
- сколько объектов в динамической памяти (куча);
- какие итоговые значения в переменных;
- что выводится в консоль.

Также код был проверен запуском/сборкой.

## 1) Присваивание `int`

```csharp
int a = 5;
int b = 6;
a = b;
b = 7;
Console.WriteLine(a);
```

Разбор:
- Выполняется по порядку сверху вниз.
- Во временной памяти выделяются 2 ячейки: `a`, `b`.
- `a = 5`, затем `b = 6`.
- `a = b` копирует значение `6` из `b` в `a`.
- `b = 7` меняет только `b`.
- В динамической памяти новые объекты не создаются.
- Итог: `a = 6`, `b = 7`.
- В консоль печатается: `6`.

Проверка запуском: вывод `6`.

## 2) Вычисление выражения

```csharp
int a = 5;
int b = a + 6;
a = 7;
Console.WriteLine(b);
```

Разбор:
- Во временной памяти: 2 ячейки (`a`, `b`).
- `a = 5`.
- `b = a + 6` => `b = 11`.
- `a = 7` не влияет на уже записанное `b`.
- В динамической памяти новых объектов нет.
- Итог: `a = 7`, `b = 11`.
- В консоль печатается: `11`.

Проверка запуском: вывод `11`.

## 3) Ссылочный тип `string`

```csharp
string a = "1";
string b = a;
a = "2";
Console.WriteLine(a);
Console.WriteLine(b);
```

Разбор:
- Во временной памяти: 2 ячейки-ссылки (`a`, `b`).
- `"1"` и `"2"` — строковые объекты (в куче/пуле интернирования строк).
- `a` сначала ссылается на `"1"`.
- `b = a` => `b` ссылается на тот же объект `"1"`.
- `a = "2"` => `a` начинает ссылаться на `"2"`, `b` остается на `"1"`.
- Итог: `a = "2"`, `b = "1"`.
- В консоль печатается:
  - `2`
  - `1`

Проверка запуском: вывод `2`, затем `1`.

## 4) Ошибка типов

```csharp
string a = 5;
```

Разбор:
- Код не компилируется.
- Ошибка: `CS0029: Cannot implicitly convert type 'int' to 'string'`.
- До выполнения программа не доходит: память под выполнение и вывод консоли не формируются.

Проверка сборкой: получена ошибка `CS0029`.

## 5) Повторное объявление переменной

```csharp
int a = 5;
int a = 6;
```

Разбор:
- Код не компилируется.
- Ошибка: `CS0128: A local variable or function named 'a' is already defined in this scope`.
- Программа не запускается, вывода в консоль нет.

Проверка сборкой: получена ошибка `CS0128`.

## Итог
- Для корректных примеров подтверждены выводы: `6`, `11`, затем `2` и `1`.
- Для некорректных примеров подтверждены ошибки компиляции: `CS0029` и `CS0128`.
